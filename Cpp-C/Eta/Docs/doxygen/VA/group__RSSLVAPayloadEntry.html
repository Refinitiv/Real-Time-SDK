RSSL Value Add Cache Payload Entry
RSSL Value Add Payload Cache

A cache Payload Entry is a container for one of the supported OMM payload types that can be stored in the cache: Field List, Element List, Map, Series, Vector, Filter List.

RSSL_VA_API RsslRet rsslPayloadEntryApply ( RsslPayloadEntryHandle  handle,
RsslDecodeIterator *  dIter,
RsslMsg *  msg,
RsslCacheError *  errorInfo 
)

When this function returns an error ( < RSSL_RET_SUCCESS), some or all of the data could not be cached. Some errors returned by this function can be handled as warnings, depending on application requirements. When the function returns an error code, the RsslCacheError.rsslErrorId can be tested for a warning indication. For errors, the rsslErrorId will have the same value as the function return value. For warnings, rsslErrorId will be set to RSSL_RET_SUCCESS. The RsslCacheError.text can be reviewed for further explanation about the errors or warnings. Examples of warning conditions: data truncated for exceeding size allocated in the field dictionary; fields in an RsslUpdateMsg ignored if they were not in the RsslRefreshMsg.

Parameters:
handleThe payload entry reference handle
dIterThe decode iterator positioned at the payload of the message. This iterator will be used for decoding as the OMM data is written to the cache entry.
msgThe partially decoded message structure
errorInfoThe error information structure will be populated if the payload data from the message could not be written to the cache entry
Returns:
Returns failure codes if the data could not be applied to the cache entry

RSSL_VA_API RsslRet rsslPayloadEntryRetrieve ( RsslPayloadEntryHandle  handle,
RsslEncodeIterator *  eIter,
RsslPayloadCursorHandle  cursorHandle,
RsslCacheError *  errorInfo 
)

This function encodes data from the cache entry into the buffer provided with the eIter parameter. The retrieval process will attempt to encode the entire container into the given buffer, but also supports multi-part retrieval for OMM containers that support fragmentation. (Note that FieldList and ElementList containers must be retrieved in a single part). If the application does not wish to use fragmentation, it should pass a null RsslPayloadCursorHandle.

For multi-part retrieval, an RsslPayloadCursorHandle is required. The application will call this function to retrieve fragments of the cached container until the cursor indicates the final part has been retrieved (rsslPayloadCursorIsComplete). The size of the buffer provided for retrieval will determine the number of fragments needed to encode the entire container.

Parameters:
handleThe payload entry reference handle
eIterThe OMM payload data in the cache entry will be encoded into the buffer associated with this iterator.
cursorHandleThe cursor handle is required to support multi-part (fragmented) retrieval of the OMM data in the cache entry. Clear the cursor (rsslPayloadCursorClear) prior to the first retrieval from the payload entry. The cursor is not needed for single part retrieval.
errorInfoThe error in formation structure will be populated if payload entry cache data could not be retrieved.
Returns:
Failure codes ( < RSSL_RET_SUCCESS) if data could not be retrieved from the container. RSSL_RET_BUFFER_TOO_SMALL indicates that the buffer size should be increased in order to retrieve the data from the entry (single part or multi-part). RSSL_RET_SUCCESS is returned when data is successfully encoded into the buffer, or if there is no data to retrieve from the entry. Note that for multi-part retrieval, RSSL_RET_SUCCESS only indicates the successful encoding of the current fragment. Check the state of the RsslPayloadCursorHandle to determine if the current fragment is the final part.
