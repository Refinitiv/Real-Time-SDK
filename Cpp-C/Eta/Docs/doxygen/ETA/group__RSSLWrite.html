RSSL_API RsslRet rsslFlush ( RsslChannel *  chnl,
RsslError *  error 
)

Typical use:
rsslFlush pushes the data in the write buffer out to the network. This should be called when write returns that there is data to flush. Under certain circumstances, write will automatically flush data.

Parameters:
chnlRSSL Channel to attempt flush on
errorRSSL Error, to be populated in event of an error
Returns:
RsslRet RSSL return value or the number of bytes pending flush
RSSL_API RsslRet rsslPing ( RsslChannel *  chnl,
RsslError *  error 
)

Typical use:
rsslPing is called to send some type of ping or heartbeat message. rsslPing will send only the message header across the network. This helps reduce overhead on the network, and does not incur any cost of parsing or assembling a special ping message type. It is the user's responsibility to send the ping message in the correct time frame. Since it is assumed a ping or heartbeat is only sent when other writing is not taking place, rsslFlush is automatically called once. The return value will be the number of bytes left to flush.

Parameters:
chnlRSSL Channel to send ping or heartbeat on
errorRSSL Error, to be populated in event of an error
Returns:
RsslRet RSSL return value or the number of bytes pending flush
RSSL_API RsslRet rsslWrite ( RsslChannel *  chnl,
RsslBuffer *  buffer,
RsslWritePriorities  rsslPriority,
RsslUInt8  writeFlags,
RsslUInt32 *  bytesWritten,
RsslUInt32 *  uncompressedBytesWritten,
RsslError *  error 
)

Typical use:
rsslWrite is called after buffer is populated with a message. This message is then written to the channels internal write buffer. If write is successful, the passed in buffer will be released automatically. In the event of a failure the user needs to call rsslReleaseBuffer. In the success case, rsslWrite will return the number of bytes pending flush.

Note:
Data is not written across the network until rsslFlush is called.
Parameters:
chnlRSSL Channel to write to
bufferBuffer to write to the network
rsslPriorityPriority to flush the message (high, medium, or low)
writeFlagsFlags for writing the buffer (RsslWriteFlags)
bytesWrittenReturns total number of bytes written.
uncompressedBytesWrittenReturns number of bytes written before compression.
errorRSSL Error, to be populated in event of an error
Returns:
RsslRet RSSL return value, RsslReturnCodes value, or the number of bytes pending flush
