RsslAckMsg Struct Reference
RsslAckMsg Structure, Initializers, and Flag values

The RsslAckMsg is used to acknowledge success/failure or completion of an RsslPostMsg or RsslCloseMsg. More...

#include <rsslAckMsg.h>

Data Fields

RsslUInt32 ackId
 ID used to associate this Ack with the message it is acknowledging.
RsslBuffer extendedHeader
 Extended header attributes, used for per-domain defined attributes that are not used when determining stream uniqueness. Presence is indicated by RsslAckFlags::RSSL_AKMF_HAS_EXTENDED_HEADER.
RsslUInt16 flags
 Flag values are used to indicate optional member presence. The available options are defined by values present in RsslAckFlags.
RsslMsgBase msgBase
 Common message header members (streamId, domainType, msgKey, etc.), msgKey is optional and typically not sent as streamId can be used to match updates to a specific item stream.
RsslUInt8 nakCode
 This enumeration indicates the reason for a negative acknowledgement(NAK). See RsslNakCodes for values. Presence is indicated by RsslAckFlags::RSSL_AKMF_HAS_NAK_CODE.
RsslUInt32 seqNum
 Sequence number intended to help with temporal ordering. Typically, this will be incremented with every message, but may have gaps depending on the sequencing algorithm being used. Presence is indicated by RsslAckFlags::RSSL_AKMF_HAS_SEQ_NUM.
RsslBuffer text
 This provides additional information about the acceptance or rejection of the message being acknowledged.
