Overview: Transport API Primitive and Container Type Enumerations

Detailed Description

This section describes each Transport API primitive and container type enumerated value and refers to each types specific section. These enumerated values are used to indicate which type is being sent or received.

Typedefs

typedef RsslUInt8 RsslContainerType
 An 8 bit value that uses an RsslDataTypes enumerated value to specify the type of content. This can be used to specify only container types.
typedef RsslUInt8 RsslDataType
 An 8 bit value that uses an RsslDataTypes enumerated value to specify the type of content. This can be used to specify a primitive or container type.
typedef RsslUInt8 RsslPrimitiveType
 An 8 bit value that uses an RsslDataTypes enumerated value to specify the type of content. This can be used to specify only primitive types.

Enumeration Type Documentation

enum RsslDataTypes
Enumerator:
RSSL_DT_BASE_PRIMITIVE_MIN 

Minimum allowed value for primitive types - used for internal UPA range checking

RSSL_DT_UNKNOWN 

(0) Unknown Data Type. This is only valid when decoding an RsslFieldEntry type that requires a dictionary look-up. If content is set defined, actual type enum will be present.
(1) Reserved
(2) Reserved

RSSL_DT_INT 

(3) Signed Integer, represents a value of up to 63 bits along with a one bit sign. See RsslInt Reference Group for more information.

RSSL_DT_UINT 

(4) Unsigned Integer, represents an unsigned value with precision of up to 64 bits. See RsslUInt Reference Group for more information.

RSSL_DT_FLOAT 

(5) 32 Bit Floating Point, represents the same range of values allowed with the system float type. See RsslFloat Reference Group for more information.

RSSL_DT_DOUBLE 

(6) 64 Bit Floating Point, represents the same range of values allowed with the system double type. See RsslDouble Reference Group for more information.
(7) Reserved

RSSL_DT_REAL 

(8) Real Numeric, a bandwidth optimized representation of a decimal or fractional value which typically requires less bytes on the wire than float or double types. See RsslReal Reference Group for more information.

RSSL_DT_DATE 

(9) Date type, contains month, day, and year values. See RsslDate Reference Group for more information.

RSSL_DT_TIME 

(10) Time type, contains hour, minute, second, millisecond, microsecond, and nanosecond values. See RsslTime Reference Group for more information.

RSSL_DT_DATETIME 

(11) DateTime type, contains all members of RSSL_DT_DATE and RSSL_DT_TIME. See RsslDateTime Reference Group for more information.

RSSL_DT_QOS 

(12) Quality of Service, represents quality of service information such as data timeliness (e.g. real time) and rate (e.g. tick by tick). See RsslQos Reference Group for more information.

RSSL_DT_STATE 

(13) State type, represents data and stream state information. See RsslState Reference Group for more information.

RSSL_DT_ENUM 

(14) Enumeration type, represents an enumeration type, defined as an unsigned two byte value. See RsslEnum Reference Group for more information.

RSSL_DT_ARRAY 

(15) Array type, represents a list of a simple primitive types that can be variable or fixed length. See RsslArray Reference Group for more information.

RSSL_DT_BUFFER 

(16) Buffer type, represents a raw byte buffer type with a char* and an RsslUInt32 length. See RsslBuffer Reference Group for more information.

RSSL_DT_ASCII_STRING 

(17) ASCII String, uses the RsslBuffer type for encoding and decoding. This enum indicates that the content is an ASCII string and should only contain characters that are valid within the ASCII specification. See RsslBuffer Reference Group for more information.

RSSL_DT_UTF8_STRING 

(18) UTF8 String, uses the RsslBuffer type for encoding and decoding. This enum indicates that the content is a UTF8 string and should only contain characters that are valid within the UTF8 encoding standard. See RsslBuffer Reference Group for more information.

RSSL_DT_RMTES_STRING 

(19) RMTES String, uses the RsslBuffer type for encoding and decoding. This enum indicates that the content is a Reuters Multilingual Text Encoding Standard string and should only contain characters that are valid within the RMTES encoding standard. See RsslBuffer Reference Group and RMTES Decoder Reference Group for more information.

RSSL_DT_VECTOR 

(136) Vector container type, used to represent index - container type paired entries.

RSSL_DT_MAP 

(137) Map container type, used to represent primitive type key - container type paired entries.

RSSL_DT_SERIES 

(138) Series container type, represents row based tabular information where no specific indexing is required.

RSSL_DT_MSG 

(141) RsslMsg container type. This can be used to nest a message inside another message (e.g. RsslPostMsg containing an RsslUpdateMsg) or nesting a message inside of another container entry (e.g. RsslMapEntry contains an RsslMsg).

RSSL_DT_JSON 

(142) JSON formatted data, use Non-RWF type encoders.

RSSL_DT_CONTAINER_TYPE_MAX 

Maximum supported container type value for this release - used for internal UPA range checking

RSSL_DT_MAX_RESERVED 

(224) Maximum Thomson Reuters reserved value. Values beyond this can be user defined types

RSSL_DT_LAST 

(255) Maximum allowed enumeration value - used for internal UPA range checking.

RSSL_API RsslUInt32 rsslPrimitiveTypeSize ( const RsslDataType  dataType)

Checks if the passed in type is a valid primitive type (0 - 128)

Parameters:
dataTypeRsslDataTypes enumerated value to return encoded size for
See also:
RsslDataTypes, RsslDataType
Returns:
RsslUInt32 containing the maximum encoded length when this can be determined. 255 (0xFF) is returned for types that contain buffer content and the length varies based on the RsslBuffer::length. 0 is returned for invalid types.
