RsslElementList Struct Reference
RsslElementList Structure, Initializers, Flag values, and Helpers

The RsslElementList container type allows the user to represent a collection of name - type - value triples, where the value can be a container type or a primitive type. The name and type information is similar to the content that would be cross referenced with a field dictionary when using an RsslFieldList. No dictionary is required to decode RsslElementEntry contents. More...

#include <rsslElementList.h>

Data Fields

RsslInt16 elementListNum
 When present, contains the identifier for an element list template, also known as a record template. Presence is indicated by RsslElementListFlags RSSL_ELF_HAS_ELEMENT_LIST_INFO. This information is typically used by caching components to determine the space needed to cache all possible content.
RsslBuffer encEntries
 Raw encoded payload for the RsslElementList. Standard encoded content presence is indicated by RsslElementListFlags RSSL_ELF_HAS_STANDARD_DATA.
RsslBuffer encSetData
 Raw encoded set defined data content. When encoding, this can be used to provide pre-encoded set data. When decoding, this will contain all encoded set data. Set data presence is indicated by RsslElementListFlags RSSL_ELF_HAS_SET_DATA.
RsslUInt8 flags
 Flag values used to indicate optional member presence or behavior. The available options are defined by values present in RsslElementListFlags.
RsslUInt16 setId
 When present, contains the set data identifier, referring to the set definition needed to decode any set defined content. Presence is indicated by RsslElementListFlags RSSL_ELF_HAS_SET_ID. When not present, value should be assumed to be 0.
