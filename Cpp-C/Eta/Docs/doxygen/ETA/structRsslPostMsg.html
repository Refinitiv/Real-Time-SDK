RsslPostMsg Struct Reference
RsslPostMsg Structure, Initializers, and Flag values

The RsslPostMsg is used to contribute content from a consumer into the platform. The post message payload can contain another OMM message, OMM container, or other opaque content. More...

#include <rsslPostMsg.h>

Data Fields

RsslBuffer extendedHeader
 Extended header attributes, used for per-domain defined attributes that are not used when determining stream uniqueness. Presence is indicated by RsslPostFlags::RSSL_PSMF_HAS_EXTENDED_HEADER.
RsslUInt16 flags
 Flag values used to indicate optional member presence and/or stream behavior. The available options are defined by values present in RsslPostFlags.
RsslMsgBase msgBase
 Common message header members (streamId, domainType, msgKey, etc.). The msgKey is optional for on-stream posting, as the streamId is used to identify the stream. However, for off-stream posting, the msgKey is typically required.
RsslUInt16 partNum
 Part number is a 15-bit value typically used with multi-part refresh messages, value should start with 0 to indicate the initial part, each subsequent part in a multi-part refresh message should increment the previous partNum by 1. Presence is indicated by RsslPostFlags::RSSL_PSMF_HAS_PART_NUM.
RsslBuffer permData
 Contains permission authorization information for this RsslPostMsg. Presence is indicated by RsslPostFlags::RSSL_PSMF_HAS_PERM_DATA.
RsslUInt32 postId
 This is a consumer-assigned identifier that is used by upstream devices to distinguish different post messages. Each RsslPostMsg in a multi-part post must use the same postId value. Presence is indicated by RsslPostFlags::RSSL_PSMF_HAS_POST_ID.
RsslPostUserInfo postUserInfo
 Information used to identify the user that posted this content to an upstream provider.
RsslUInt16 postUserRights
 Conveys the rights or abilities of the user posting this content. See RsslPostUserRights for the specific values and their meaning. Presence is indicated by RsslPostFlags::RSSL_PSMF_HAS_POST_USER_RIGHTS.
RsslUInt32 seqNum
 Sequence number intended to help with temporal ordering. Typically, this will be incremented with every message, but may have gaps depending on the sequencing algorithm being used. Presence is indicated by RsslPostFlags::RSSL_PSMF_HAS_SEQ_NUM.
