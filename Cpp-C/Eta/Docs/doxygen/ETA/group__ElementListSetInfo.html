RSSL_API RsslRet rsslDecodeElementSetDefDictionary ( RsslDecodeIterator *  dIter,
RsslElementSetDefDb *  dictionary,
RDMDictionaryVerbosityValues  verbosity,
RsslBuffer *  errorText 
)

This function may be called multiple times on the same dictionary, to load information from dictionaries that have been encoded in multiple parts.

Typical use:

  1. Initialize the RsslFieldSetDefDb using rsslClearFieldSetDb, then allocate the definitions pointer array and version information with rsslAllocateElementSetDefDb.
  2. When a response message containing global field set definition data is received, call rsslDecodeElementSetDefDictionary.
  3. Continue with decoding messages.
  4. When finished with the global set definition dictionary, call rsslDeleteElementSetDefDb to free all allocated memory.
    Parameters:
    dIterAn iterator to use. Must be set to the encoded buffer.
    dictionaryThe dictionary to store the decoded field information.
    verbosityThe desired verbosity to decode. See RDMDictionaryVerbosityValues.
    errorTextBuffer to hold error text if the decoding fails.
    See also:
    RsslElementSetDefDb, RDMDictionaryVerbosityValues, rsslEncodeElementSetDefDictionary

Typical use:

  1. Call rsslDecodeMap(), rsslDecodeSeries() or rsslDecodeVector()
  2. Call rsslDecodeLocalElementSetDefDb()

If decoding local set definitions from a single thread, RSSL has internal memory that will be
decoded into. To use this, entries.data and entries.length should be 0.
If decoding local set definitions from across multiple threads, the user should supply memory
by populating entries.data. entries.length should be set to the number of bytes available in data.
Alternatively, a separate iterator may be set to the container's encSetDefs and passed into rsslDecodeLocalElementSetDefDb() (the appropriate version information should also be set).

Parameters:

RSSL_API RsslRet rsslEncodeElementSetDefDictionary ( RsslEncodeIterator *  eIter,
RsslElementSetDefDb *  dictionary,
int *  currentSetDef,
RDMDictionaryVerbosityValues  verbosity,
RsslBuffer *  errorText 
)

This function supports building the encoded data in multiple parts -- if there is not enough available buffer space to encode the entire dictionary, subsequent calls can be made to this function, each producing the next segment of fields.

Typical use:

  1. Initialize the RsslElementSetDefDb using rsslClearElementSetDb, then allocate the definitions pointer array and version information with rsslAllocateElementSetDefDb.
  2. Add the set definitions to the RsslElementSetDefDb using rsslAddElementSetDefToDb.
  3. Start encoding the message that contains this dictionary payload.
  4. Call rsslEncodeElementSetDefDictionary.
  5. If rsslEncodeElementSetDefDictionary returns RSSL_RET_DICT_PART_ENCODED, go back to step 3, and encode the next part of the payload. Otherwise, if RSSL_RET_SUCCESS is returned, continue.
  6. When finished with the global set definition dictionary, call rsslElementFieldSetDefDb to free all allocated memory.
    Parameters:
    eIterIterator to be used for encoding. Prior to each call, the iterator must be cleared and initialized to the buffer to be used for encoding.
    dictionaryThe dictionary to encode global element set definition information from.
    currentSetDefTracks which set definitions have been encoded in case of multi-part encoding. Must be initialized to 0 on the first call and is updated with each successfully encoded part.
    verbosityThe desired verbosity to encode. See RDMDictionaryVerbosityValues.
    errorTextBuffer to hold error text if the encoding fails.
    Returns:
    RSSL_RET_DICT_PART_ENCODED when encoding parts, RSSL_RET_SUCCESS for final part or single complete payload.
    See also:
    RsslElementSetDefDb, RDMDictionaryVerbosityValues, rsslDecodeElementSetDefDictionary
