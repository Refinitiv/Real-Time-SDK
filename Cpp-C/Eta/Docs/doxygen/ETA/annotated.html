Data Structures
Here are the UPA C Edition data structures with brief descriptions:
RsslAcceptOptionsRSSL Accept Options used in the rsslAccept call
RsslAckMsgThe RsslAckMsg is used to acknowledge success/failure or completion of an RsslPostMsg or RsslCloseMsg
RsslArrayThe RsslArray is a uniform primitive type that can contain multiple simple primitive entries. Entries can be either fixed length or variable length
RsslBindOptionsRSSL Bind Options used in the rsslBind call
RsslChannelRSSL Channel structure returned via rsslAccept() or rsslConnect() calls
RsslChannelInfoRSSL Channel Info returned by rsslGetChannelInfo call
RsslCloseMsgThe RsslCloseMsg is used to indicate that a consuming application is no longer interested in content on the stream being closed
RsslComponentInfoConnected Component Information, used to identify components from across the connection
RsslConnectionInfoOptions used for configuring the network connection
RsslConnectOptionsRSSL Connect Options used in the rsslConnect call
RsslDataDictionaryA data dictionary Houses all known fields loaded from a field dictionary and their corresponding enum types loaded from an enum type dictionary. The dictionary also saves general information about the dictionary itself -- this is found in the "!tag" comments in the file or in the summary data of dictionaries encoded via the official domain model
RsslDateThe RsslDate type allows for bandwidth optimized representation of a date value containing month, day, and year information
RsslDateTimeThe RsslDateTime type allows for bandwidth optimized representation of both a date and a time value containing month, day, year, hour, minute, second, millisecond, microsecond, and nanosecond information
RsslDecIteratorIterator used for RWF decoding
RsslDecIterLevel
RsslDictionaryEntryA data dictionary entry, containing field information and an enumeration table reference if present
RsslElementEntryOne entry contained in an RsslElementList. The RsslElementEntry::dataType indicates the content type
RsslElementListThe RsslElementList container type allows the user to represent a collection of name - type - value triples, where the value can be a container type or a primitive type. The name and type information is similar to the content that would be cross referenced with a field dictionary when using an RsslFieldList. No dictionary is required to decode RsslElementEntry contents
RsslElementSetDefElement List Set Definition
RsslElementSetDefDbGlobal Element List Set Definitions Database
RsslElementSetDefEntryElement List Set Element Encoding
RsslEncIteratorIterator used for RWF encoding
RsslEncLevel
RsslEncodeSizeMark
RsslEncryptionOptsOptions used for configuring an encrypted tunneled connection (RSSL_CONN_TYPE_ENCRYPTED). see rsslConnect see RsslConnectOptions
RsslEnumTypeA single defined enumerated value
RsslEnumTypeTableA table of enumerated types. A field that uses this table will contain a value corresponding to an enumerated type in this table
RsslErrorRSSL Error message
RsslFieldEntryOne entry contained in an RsslFieldList. When encoding, RsslFieldEntry::dataType should be specified. When decoding set data, RsslFieldEntry::dataType will be populated with information from set definition. When decoding and RsslFieldEntry::dataType is RSSL_DT_UNKNOWN, user should cross reference RsslFieldEntry::fieldId with the appropriate field dictionary to determine type and name information
RsslFieldListThe RsslFieldList container type allows the user to represent a collection of field identifier - value pairs, where the value can be a container type or a primitive type. The field identifier (RsslFieldEntry::fieldId) can be cross referenced with a field dictionary to determine the specific type of the content. The dictionary contains specific name and type information (e.g. RDMFieldDictionary)
RsslFieldSetDefField List Set Definition
RsslFieldSetDefDbGlobal Field List Set Definitions Database
RsslFieldSetDefEntryField List Set Definition Field Entry
RsslFilterEntryOne entry contained in an RsslFilterList. If RsslFilterEntry::flags contains RSSL_FTEF_HAS_CONTAINER_TYPE, the contents of the entry is specified by RsslFilterEntry::containerType; otherwise the contents of the entry is specified by RsslFilterList::containerType. Any contents should be applied following the rules of the specified RsslFilterEntry::action ( RsslFilterEntryActions)
RsslFilterListThe RsslFilterList is a non-uniform container type of filterId (8-bit identifier) – value pair entries. Each entry, known as an RsslFilterEntry, contains an id corresponding to one of 32 possible bit-value identifiers. These identifiers are typically defined by a domain model specification and can be used to indicate interest or presence of specific entries through the inclusion of the of the id in the RsslMsgKey::filter
RsslGenericMsgThe RsslGenericMsg allows applications to bidirectionally send messages without any implied interaction semantics (e.g. no Request/Response paradigm)
RsslInitializeExOptsStructure that provides additional initialization information for the RSSL API
RsslInProgInfoRsslInProgInfo Information for the In Progress Connection State If a backward compatibility reconnection occurs, the file descriptor may change. This is how that information is relayed
rsslJITOptsJust-In-Time loaded library configuration options
RsslLibraryVersionInfoLibrary Version Information structure to be populated with library version info
RsslLocalElementSetDefDbLocal Message Element List Set Definitions Database, must be cleared before use
RsslLocalFieldSetDefDbLocal Message Field List Set Definitions Database. Must be cleared before use
RsslMapThe RsslMap is a uniform container type of associated key–container pair entries. Each entry, known as an RsslMapEntry, contains an entry key and a value. The key is a base primitive type as indicated by RsslMap::keyPrimitiveType and the value is a container type as indicated by RsslMap::containerType
RsslMapEntryOne entry contained in an RsslMap. The contents of the entry is specified by RsslMap::containerType and the key's type is specified by RsslMap::keyPrimitiveType. Any contents should be applied following the rules of the specified RsslMapEntry::action ( RsslMapEntryActions)
RsslMCastOptsOptions used for configuring multicast specific transport options (RSSL_CONN_TYPE_RELIABLE_MCAST)
RsslMCastStatsMulticast statistics returned by rsslGetChannelInfo call
RsslMsgThe OMM Message Union. All message encoding and decoding functions expect the RsslMsg type. Any specific message class can be cast to the RsslMsg, and an RsslMsg can be cast to any specific message class. When decoding, the RsslMsg.msgBase contains common members that can be used to identify the specific message class or domain type
RsslMsgBaseThe RsslMsgBase contains common members that are used across all Rssl Messages
RsslMsgKeyThe RsslMsgKey contains a variety of attributes used to identify the contents flowing within a particular stream. This information, in conjunction with domainType and quality of service information, can be used to uniquely identify a data stream
RsslNodeId
RsslPostMsgThe RsslPostMsg is used to contribute content from a consumer into the platform. The post message payload can contain another OMM message, OMM container, or other opaque content
RsslPostUserInfoThe RsslPostUserInfo is used to identify the user that has posted a message
RsslProxyOpts
RsslQosThe RsslQos can be used to indicate the quality of service associated with content. This includes timeliness (e.g. age) and rate (e.g. period of change) information
RsslReadInArgsRsslReadEx input arguments
RsslReadOutArgsRsslReadEx output arguments
RsslRealThe RsslReal allows the user to represent fractional or decimal values, with minimal conversion overhead, by using an RsslInt and a format hint
RsslRefreshMsgThe RsslRefreshMsg is provided as an initial response or when an upstream source requires a data resynchronization point
RsslRequestMsgThe RsslRequestMsg is used to provide an initial or subsequent data synchronization point, including quality of service and payload content. When used to respond to an RsslRequestMsg, a solicited RsslRefreshMsg can be used. If pushing out a content change to downstream applications, an unsolicited RsslRefreshMsg can be used. When the stream is instantiated because of an RsslRequestMsg, the streamId from the request should be used; when a provider instantiates the stream with an RsslRefreshMsg (e.g. Non-Interactive provider), a negative streamId should be used
RsslRmtesCacheBufferRsslRmtesCacheBuffer structure
RsslSegmentedNetworkOptions used for configuring a segmented network (seperate send and receive networks)
RsslSeqMCastOptsOptions used for configuring sequenced multicast specific transport options (RSSL_CONN_TYPE_SEQ_MCAST). see rsslConnect see RsslConnectOptions
RsslSeriesThe RsslSeries is a uniform container type entries. Each entry, known as an RsslSeriesEntry, is implicitly indexed and houses a container type payload as indicated by RsslSeries::containerType. This is well suited for sending infrequently or non-updating table like content, where each RsslSeriesEntry represents an additional row in a table
RsslSeriesEntryOne entry contained in an RsslSeries. The contents of the entry is specified by RsslSeries::containerType. RsslSeriesEntry has an implicit action of Add, so any additional entries should be appended to any existing content
RsslServerRSSL Server structure returned via the rsslBind call
RsslServerInfoRSSL Server Info returned by rsslGetServerInfo call
RsslShmemOptsOptions used for configuring shared memory specific transport options (RSSL_CONN_TYPE_UNIDIR_SHMEM)
RsslStateState structure, contains information used to convey data and stream health information
RsslStatusMsgThe RsslStatusMsg is used to convey state information, permission change, or group Id change information on a stream. An interactive or non-interactive providing application can also close a stream with a status message
RsslTcpOptsOptions used for configuring TCP specific transport options (RSSL_CONN_TYPE_SOCKET, RSSL_CONN_TYPE_ENCRYPTED, RSSL_CONN_TYPE_HTTP)
RsslTimeThe RsslTime type allows for bandwidth optimized representation of a time value containing hour, minute, second, millisecond, microsecond, and nanosecond information
RsslTraceOptionsRSSL Trace Options structure used to configure XML Tracing
RsslU16BufferRsslU16Buffer structure
RsslUnifiedNetworkOptions used for configuring a unified/fully connected mesh network
RsslUpdateMsgThe RsslUpdateMsg is used by providers(both interactive and non-interactive) to convey changes to data associated with an item stream
RsslVectorThe RsslVector is a uniform container type of associated index–container pair entries. Each entry, known as an RsslVectorEntry, contains an RsslUInt index and a container payload. The payload is a container type as indicated by RsslVector::containerType
RsslVectorEntryOne entry contained in an RsslVector. The contents of the entry is specified by RsslVector::containerType. Any contents should be applied following the rules of the specified RsslVectorEntry::action ( RsslVectorEntryActions)
RsslWriteInArgsRsslWriteEx input arguments
RsslWriteOutArgsRsslWriteEx output arguments
RwfBufferRwfBuffer type
