TREP Authentication
The Transport API can use the TREP Authentication feature, which provides enhanced authentication functionality when used with TREP and DACS. This feature requires TREP 3.1 or later.
A consumer or non-interactive provider application can pass a token generated from a token generator based on the user's credentials to TREP. TREP passes this token to a local token authenticator for verification.
The token must be encoded in the initial login RsslRequestMsg with:
•
msgKey.Name set to one byte of 0x00, and
•
msgKey.NameType set to RDM_LOGIN_USER_AUTHN_TOKEN.
The token will be in the msgKey.attrib's RsslElementList, with an RsslElementEntry named authenticationToken.
For additional information, refer to the Transport API RDM Usage Guide for encoding and decoding Login messages, and the TREP Authentication User Manual1 for details on setting up TREP and the token generator.
1
For further details on TREP Authentication, refer to the TREP Authentication User Manual, accessible on Thomson Reuters MyAccount in the DACS product documentation set.
