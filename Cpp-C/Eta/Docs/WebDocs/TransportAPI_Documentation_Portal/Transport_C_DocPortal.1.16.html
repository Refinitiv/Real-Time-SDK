Transport API C Edition

Product Features
General Functionality
The Transport API provides the following functions:
•
Ability to fragment and reassemble large messages.
•
Ability to pack multiple small messages into the same network buffer.
•
Ability to choose a thread-locking model based on application needs. This allows clients to develop single-threaded, multi-threaded thread safe, and thread-aware solutions.
•
Full control of the number of available message buffers, as well as the ability to change that number during runtime.
•
No configuration, message file, or log file dependencies: everything is programmatic.
•
Allows users to send messages with different priority levels.
•
Support for the following transport configurations:
-
TCP socket
-
Reliable UDP Multicast
-
Unreliable, Sequenced UDP Multicast
-
HTTP
-
HTTPS
-
Shared Memory
