upacConsPerf
The upacConsPerf application measures the performance of the Transport API and its encoders and decoders in consuming Level I Market Price content (either directly from an OMM provider or via TREP).
To measure latency, a timestamp is randomly placed in each burst of updates by the provider. The consumer then decodes the timestamp from the update to determine end-to-end latency. upacConsPerf also supports measurement of posting latency and can measure performance using the base Transport, the Reactor, or the Reactor and watch list.

upacNIProvPerf
The upacNIProvPerf application measures performance of the Transport API and its encoders and decoders in providing Level I Market Price content (to an Advanced Data Hub (ADH)). You can configure upacNIProvPerf to measure the performance of the base Transport API or the Reactor.
The provider creates two types of threads:
•
The main thread, which collects and records statistical information.
•
Provider threads, each of which connect to an ADH and provide market data.

upacProvPerf
The upacProvPerf application measures the performance of the Transport API and its encoders and decoders in providing Level I Market Price content (either directly to an OMM Consumer or to a TREP component).
You can configure the provider to provide updates at various rates. To measure latency, a timestamp is placed randomly in each burst of updates. The consumer then decodes the timestamp from the update to determine the end-to-end latency. You can configure upacProvPerf to measure the performance of the base Transport API or the Reactor.
Click here to access the readme.
upacTransportPerf
The upacTransportPerf application measures the performance of all connection types supported by the Transport API using variable message sizes (including shared memory, reliable multicast, TCP/IP, and HTTP).
