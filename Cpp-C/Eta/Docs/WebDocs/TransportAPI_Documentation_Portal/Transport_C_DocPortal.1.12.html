Transport API C Edition

PDFs and Reference Manuals
•
Transport API C Edition Developers Guide
The developers guide includes information and examples that aid in developing applications using the Transport API. It provides detailed descriptions of OMM Data and Message package concepts and use of the RSSL Reliable Transport.
•
Transport API C Edition RDM Usage Guide
The RDM Usage Guide describes the content layout and use for Thomson Reuters-provided domain message models. The guide describes content available through the Thomson Reuters Enterprise Platform, EleKtron, Reuters Data Feed Direct, and various other sources.
•
Transport API C Edition Reference Guide
This is a doxygen-generated reference guide, created in part from the comments and usage information provided in the Transport API C Edition header files. The content herein supplements the developers guide.
•
Transport API C Edition Value Added Components Developers Guide
This developers guide provides information and examples that aid in application development using Value Added Components (including details on the Transport API Reactor and Domain Representation helpers).
•
Transport API C Edition Value Added Components Reference Manual
This is a doxygen-generated reference guide, created in part from the comments and usage information provided in the Transport API C Edition Value Added Components header files. The Transport API Value Added Components Reference supplements the Value Added Components developers guide.
•
Transport API C Edition Open Source Performance Tools Guide
This guide documents the general design and usage of the tools provided to measure the performance of the API. It describes how features of the API send and receive data with high throughput and low latency, both with direct API connections and when the API communicates through various TREP components (i.e., the Advanced Data Hub and/or Advanced Distribution Server).
•
Transport API C Edition Training Example Guide
This guide provides a step-by-step training overview for consumer, interactive provider and non-interactive provider application types. For each application type, an example is given showing how to build an application from scratch. Each example starts at a very basic level and then adds functionality with each new step. Examples are meant to simulate a classroom environment where an instructor starts with basic concepts and builds upon those concepts resulting in a basic, functional application.
•
DACS Lock Developers Guide
This document describes how to create DACS Locks and manipulate permission information. The DACS Lock package is made available as a supplement to the Transport API so applications can properly create permission data for the content they provide.
•
ANSI Page Developers Guide
This document describes ANSI Page encoding and decoding. The ANSI package is made available as a supplement to the Transport API so, if needed, an application can provide or consume ANSI Page-based content.
