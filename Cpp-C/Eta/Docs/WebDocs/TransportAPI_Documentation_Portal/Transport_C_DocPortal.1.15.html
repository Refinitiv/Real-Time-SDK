Transport API C Edition

Overview
The Transport API C Edition is a low-level API that provides the an extremely flexible development environment for application developers. It is the foundation on which all Thomson Reuters Open Message Model (OMM) -based components are built. By utilizing an API at the transport level (cf. the OSI model), a client can write to the same API used by TREP’s Core Infrastructure and achieve the same levels of performance. The Transport API is thread safe and thread aware. It is capable of handling all OMM primitives and Domain Models, including Thomson Reuters’s models as well as user-defined models.
The Transport API is composed of several packages, each of which comprises all the necessary components, including libraries, full documentation, and examples of how the API is used.
Value Added Components provide alternative entry points for applications to leverage the OMM-based Transport API and can simplify and compliment its use. You can use Value Added Components to choose and customize the balance between ultra-high performance raw access and ease-of-use feature functionality. The choice to leverage some or all of these components is up to the application developer as their use is optional. Value Added Components are written to the Transport API interfaces and are designed to work alongside it and have a similar look and feel.
For more information refer to the Transport API C Edition Value Added Components Developers Guide and the Transport API Value Added Components Reference Manual.
