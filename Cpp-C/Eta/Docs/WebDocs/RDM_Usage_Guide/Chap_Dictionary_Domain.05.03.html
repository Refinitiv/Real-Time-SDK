Decoding Field List Contents with Field and Enumerated Types Dictionaries
By itself, an RsslFieldEntry contains little information (only the fieldId and its encoded value in encData). With few exceptions, the type information is not sent with the data and will appear as RSSL_DT_UNKNOWN. To decode the value, the application must cross reference the fieldId with the correct Field Dictionary to determine its type. The RsslFieldList.dictionaryId can optionally convey the identifier of the associated dictionary.
Figure 6. Field List Referencing Field Dictionary
The consumer matches the fieldId of the RsslFieldEntry to the information about that FID in the dictionary. This tells the consumer the type (RsslDataTypes) of the data contained in the RsslFieldEntry. The consumer can now decode the data by calling the appropriate decode function for that type.
