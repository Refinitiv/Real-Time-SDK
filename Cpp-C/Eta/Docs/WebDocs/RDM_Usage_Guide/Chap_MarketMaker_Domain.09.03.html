Market Maker Refresh Message Use
A Market Maker refresh message is encoded and sent by OMM interactive provider and OMM non-interactive provider applications.
The Market Maker refresh can be sent in multiple parts. It is possible for update and status messages to be delivered between parts of a refresh message, regardless of streaming or non-streaming request.
Providers must set the RSSL_RFMF_CLEAR_CACHE flag on the solicited RsslRefreshMsg. For multi-part refreshes, the RSSL_RFMF_CLEAR_CACHE flag must be set on the first part only.
