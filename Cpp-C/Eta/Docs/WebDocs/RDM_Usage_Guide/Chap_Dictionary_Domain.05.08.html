Payload and Summary Data for the Refresh Message
For example layouts of some dictionary messages, refer to Section 5.11.
The payload varies depending on the type of dictionary being sent. The domain model layout for each type of dictionary is described in the following sections.
Although the structure varies for each type of dictionary, each uses an RsslElementList to identify its type, version, and DictionaryId. If sending the dictionary as a multi-part refresh, this data must be present only in the first part.
