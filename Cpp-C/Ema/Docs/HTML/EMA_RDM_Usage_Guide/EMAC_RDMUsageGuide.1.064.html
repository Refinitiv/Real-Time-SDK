Specific Usage: Enterprise Platform for Real-Time
When an Advanced Data Hub (ADH) receives a Delete MapAction for a service, it marks the service down and sends a service down state in the State FilterItem. It does not delete the service from its map. It is recommended at this time that no EMA providers send Delete MapActions to remove services.
The ADS always sends SupportsQoSRange = 0.
If SingleOpen is enabled in the login, then ADS always sends AcceptingRequests =1. If SingleOpen is disabled, the value of AcceptingRequests will always equal the value of ServiceState.
OpenWindow is not sent by the ADS. The ADH will obey the OpenWindow that a provider application sends it. If the OpenWindow is not provided, it will use the value specified in its configuration.
If a Concrete Serviceâ€™s QoS Array includes more than one Quality of Service, only the best Quality of Service is used.
Thomson Reuters recommends that you include ServiceState.Status information when providing a Directory Response with the State FilterItem to an ADH. If the ADH is configured to use SingleOpen and ServiceState.Status information is not provided in the State FilterItem, the ADH does not convey service or item status to downstream components. Thus the ADH can fail to send downstream notifications when the service becomes unavailable or items provided by the service become Suspect.
