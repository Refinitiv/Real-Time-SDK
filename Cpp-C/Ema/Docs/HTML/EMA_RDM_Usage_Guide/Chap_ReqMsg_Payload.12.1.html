A.1 View Definition
The client application can specify interest in a specific subset of fields or elements (known as a ‘View’). This is done by encoding an array of the desired fields or elements in the request message payload. The response Message will contain a list of the requested fields or elements and possibly some others depending on factors such as aggregation and the ability of the provider to supply the requested view. Unless otherwise specified, this is supported on any non-administrative RDM and any user defined DMM. For more information, refer to the Message API C++ Edition Developers Guide. When requesting a new view or changing a view, at a minimum, the request message payload contains an element list with the following entries (any default behavior is included in the element’s description):
