Channel.getBuffer Return Values
The following table defines TransportReturnCodes and Error.errorId values that can occur while using Channel.getBuffer.
 
Table 33: Channel.getBuffer TransportReturnCodes  
RETURN CODE
DESCRIPTION
Valid buffer returned
Success Case: TransportReturnCodes.SUCCESS
A TransportBuffer is returned to the user. Transport Buffer.data refers to the underlying ByteBuffer.
NULL buffer returned
Error Code: TransportReturnCodes.NO_BUFFERS
NULL is returned to the user. This value indicates that there are no buffers available to the user. See Error content for more details.
This typically occurs because all available buffers are queued and pending flushing to the connection. The application can use Channel.flush to attempt releasing buffers back to the pool (refer to Section 9.10.2). Additionally, the Channel.ioctl method can be used to increase the number of guaranteedOutputBuffers (refer to Section 9.14).
NULL buffer returned
Error Code: TransportReturnCodes.FAILURE
NULL is returned to the user. This value indicates that some type of general failure has occurred. The Channel should be closed, refer to Section 9.13. See Error content for more details.
NULL buffer returned
Error Code: TransportReturnCodes.INIT_NOT_INITIALIZED
Indicates that the Transport has not been initialized. See the Error content for more details. For information on initializing, refer to Section 9.2.
