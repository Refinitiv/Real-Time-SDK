StreamStates Values
 
Table 73: StreamStates Values  
STream State
DESCRIPTION
UNSPECIFIED
streamState is unspecified. Typically used as a structure initialization value and is not intended to be encoded or decoded.
OPEN
streamState is open. This typically means that data is streaming: as data changes, they are sent on the stream.
NON_STREAMING
streamState is non-streaming. After receiving a final RefreshMsg or StatusMsg, the stream is closed and updated data is not delivered without a subsequent re-request. Update messages might still be received between the first and final part of a multi-part refresh.
For further details, refer to Section 13.1.
CLOSED_RECOVER
streamState is closed, however data can be recovered on this service and connection at a later time. This state can occur via either a RefreshMsg or a StatusMsg.
Single Open behavior can modify this state (continuing to indicate a stream state of OPEN) and attempt to recover data on the userâ€™s behalf.
For further details on Single Open behavior, refer to Section 13.5.
CLOSED
streamState is closed. Data is not available on this service and connection and is not likely to become available, though the data might be available on another service or connection. This state can result from either an RefreshMsg or an StatusMsg.
REDIRECTED
streamState is redirected. The current stream is closed and has new identifying information. The user can issue a new request for the data using the new message key data from the redirect message. This state can result from either a RefreshMsg or a StatusMsg.
For further details, refer to Section 12.1.3.2.
