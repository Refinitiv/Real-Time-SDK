Dictionary Refresh Message
A Dictionary refresh message is encoded and sent by OMM Interactive and non-interactive provider applications and provides the consumer with the content of the requested dictionary.
 
Table 35: Dictionary Refresh Message Member Use  
COMPONENT
DESCRIPTION / VALUE
msgClass
Required. MsgClasses.REFRESH == 2
domainType
Required. DomainTypes.DICTIONARY == 5
state
Required. Contains stream and data state information for the dictionary stream.
qos
Not used.
seqNum
Optional. A user-specified, item-level sequence number which can be used by the application for sequencing messages with this stream.
groupId
Not used.
permData
Optional. Used if provided dictionary requires permissioning.
extendedHeader
Not used.
msgKey.serviceId
Required. Specifies the serviceId of the service providing the dictionary.
Note: The application should set either the ServiceName or msgKey.serviceId of the service, but not both.
msgKey.nameType
Not used.
msgKey.name
Required. Specify a msgKey.flags value of MsgKeyFlags.HAS_NAME. Populate msgKey.name with the name corresponding to the provided dictionary.
msgKey.filter
Required. The consumer should set the msgKey.filter according to how much information is needed:
•
Dictionary.VerbosityValues.INFO == 0x00: Provides version information only.
•
Dictionary.VerbosityValues.MINIMAL == 0x03: Provides information needed for caching.
•
Dictionary.VerbosityValues.NORMAL == 0x07: Provides all information needed for decoding.
•
Dictionary.VerbosityValues.VERBOSE == 0x0F: Provides all information (including comments).
Providers are not required to support the MINIMAL and VERBOSE filters.
msgKey.identifier
Not used.
msgKey.attrib
Not used.
Payload
Required. The payload structure varies depending on the dictionary’s type. However, payload is typically a Series containing a ElementList, while the series summaryData indicates the specific dictionary type.
