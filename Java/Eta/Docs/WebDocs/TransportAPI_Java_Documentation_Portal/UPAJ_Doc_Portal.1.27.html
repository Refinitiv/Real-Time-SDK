Transport API Java Edition

Running Examples
Most Transport API examples can be run directly from a Command Line prompt. All example configuration and parameters are provided as command line arguments, where the -? argument displays a list of all configuration parameters. All parameters have default values to allow basic functionality when running providers and consumers on the same machine.
For example, to run the Consumer example using its default values, you can use the following command line:
 
Consumer
 
or specify any of the following options on the command line
 
Consumer [-uname <LoginUsername>] [-h <SrvrHostname>] [-p <SrvrPortNo>] [-bc <hostname>:<port>]
[-c <hostname>:<port> <service name> <domain>:<item name>,...] [-connectionType <type>]
[-s <ServiceName>] [-view] [-post] [-offpost] [-snapshot] [-sl [<SymbolList Name>]]
[-mp|-mpps <MarketPrice ItemName>] [-mbo|-mbops <MarketByOrder ItemName>]
[-mbp|-mbpps <MarketByPrice ItemName>] [-yc|-ycps <YieldCurve ItemName>] [-runtime <seconds>]
[-proxy <host>:<port>] [-x] [-publisherInfo]
