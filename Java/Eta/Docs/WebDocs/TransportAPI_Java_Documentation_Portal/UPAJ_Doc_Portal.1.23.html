About the Transport API Performance Tool Suite
The Performance Tool Suite provides a consistent set of platform test applications that look and behave consistently across the Transport APIs, Message API, and RFA API’s. The tool suite covers the various OMM-based API products and allows Thomson Reuters’s internal and external clients to compare latency and throughput tradeoffs of the various APIs and their differing functionality sets.
All tools in the suite are buildable open-source and demonstrate best practice and coding for performance with their respective APIs. Future releases of API products will expand on these tests to include other areas of functionality (e.g., batch requesting, generic message performance, etc.). Clients can run these tools to determine performance results for their own specific environment, recreate Thomson Reuters-released performance numbers that were generated using these tools, and modify the open source to tune and tweak applications to best match their own end-to-end needs.
The Transport API Performance Tools Guide document discusses the general design and usage of the tools. It details how features of the API are used to send and receive data with high throughput and low latency. This information applies both when the API is directly connected to itself as well as through Thomson Reuters Enterprise Platform for Real-Time (TREP-RT) components, such as Advanced Data Hub (ADH) and Advanced Distribution Server (ADS).
