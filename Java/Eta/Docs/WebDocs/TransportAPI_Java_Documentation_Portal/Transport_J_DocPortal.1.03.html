Understanding the Transport API
What is the Transport API Java Edition?
The Transport API is the productized version of Thomson Reutersâ€™s own low-level, internal API. The Transport API is currently used by the Thomson Reuters Enterprise Platform (TREP) and its dependent APIs for the optimal distribution of OMM/RWF data.
Because the Transport API (also known as the RSSL API) has been in use since 2006, it is currently deployed in products such as the ADS, ADH, EMA, RFA, RDF-D, Elektron, and Eikon. Due to its well-integrated and common usage across these products, clients can use the Transport API to write applications for use on TREP with the highest performance, highest throughput, and lowest latency.
The Transport API is a custom API that supports the OMM message model and complements other APIs, allowing users to choose the types of functionality and layer (Session or Transport) they use to access the Thomson Reuters Enterprise Platform (TREP). With the breadth of offerings that Thomson Reuters provides, customers can choose between a feature-loaded session layer API (i.e., EMA) and a high-performance transport layer API (the Transport API).
