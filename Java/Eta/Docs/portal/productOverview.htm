Elektron Transport API (ETA) - Documentation Portal

Overview

Return to Top

The Elektron Transport API - Java Edition (ETAJ) lets applications achieve optimal distribution of OMM/RWF data in the Java Language. ETAJ is the strategic low-level API for multi-threaded access to subscription, publication, and contribution capabilities of Thomson Reuters Enterprise Platform. It is designed to take full advantage of throughput and latency benefits of Advanced Distribution Server, Advanced Data Hub, Data Direct Feed, and Elektron. The ETAJ fully supports all OMM constructs and messages. With the addition of the Java language to the ETA suite, users now have a choice between a session level API (RFA), and a high performance / low latency transport level API (ETAJ).

Product Features

Return to Top

ETA provides the following functionality to all applications:

  • Ability to fragment and reassemble large messages.
  • Ability to pack multiple small messages into the same network buffer.
  • Ability to choose a thread locking model based on application needs. This allows clients to develop single-threaded, multi-threaded thread safe, and thread-aware solutions.
  • Full control of the number of available message buffers, as well as the ability to change that number during runtime.
  • No configuration, message file, or log file dependencies. Everything is programmatic.
  • Allows users to send messages with different priority levels.
  • The following transport configurations are supported:
    • TCP socket
    • Reliable UDP Multicast
    • HTTP
    • HTTPS
    • Unidirectional Shared Memory

Additionally, ETA provides the following application-specific capabilities:

  • Consumer Applications:
    • Ability to make Streaming and Snapshot based subscription requests.
    • Ability to make OMM Batch, View, and Symbol List subscription requests to capable providers.
    • Support for Pause and Resume functionality on active data streams to capable providers.
    • Ability to send Post Messages used for Consumer-based Publishing and contributions to capable providers.
    • Ability to send and receive Generic Messages.
    • Ability to consume data with a tunneled HTTP connection to an ADS.
  • Interactive Provider Applications:
    • Ability to receive requests and respond to Streaming and Snapshot based subscription requests (Previously known as Managed or Sink-Driven Server applications).
    • Ability to receive and respond to requests for Batch, View, and Symbol List subscription requests.
    • Ability to receive Pause and Resume requests on active data streams.
    • Ability to receive acknowledge Post Messages used for Consumer-based Publishing and contributions.
    • Ability to send and receive Generic Messages.
    • Ability to write a server-based application that accepts multiple connections, or allows multiple consumers to connect to a provider.
  • Non-Interactive Provider Applications:
    • Ability to start up and publish data directly to ADH (Previously known as Source-Driven or Broadcast style server applications).

This release also includes the ETA Value Added components. These components are intended to simplify ETA usage, and reduce the amount of code required to create OMM-based applications. In addition to the previous ETA features, these components provide the following additional features:

  • The ETA Reactor component is a connection management and event processing component.
  • The Administration Domain Model Representations reduce the code required to interact with the RDM administrative domains.
  • The Value Added Utilities are a collection of common classes mainly used by the ETA Reactor.
ETA Documentation

Return to Top

The following documents are included in this release.

ETA Developers Guide

This document provides information and examples intended to aid in development with the Elektron Transport API product (formerly known as Ultra Performance API or UPA). It provides detailed descriptions for OMM Data and Message package concepts and use of the RSSL Reliable Transport.

ETA RDM Usage Guide

This document describes the message layout and use for Thomson Reuters provided domain message models. This content is available through the Thomson Reuters Enterprise Platform, EleKtron, Reuters Data Feed Direct, and various other sources.

ETA Reference Manual

This is a javadoc generated reference document, based on the comments and usage information provided in the ETA public interfaces. This is intended as a supplement to the ETA Developers Guide.

ETA Examples Reference Manual

This is a javadoc generated reference document, based on the comments and usage information provided in the ETA Example Applications.

ETA Value Added Components Portal

This is the main documentation portal for the ETA Value Added Components.

ETA Performance Tools Guide

This document details the ETA Open Source Performance Tool Suite, their usage, and their design and behavior.

ETA Performance Tools Reference Manual

This is a javadoc generated reference document, based on the comments and usage information provided in the ETA Performance Tools.

Utilities Documentation

Return to Top

The following documents are included in this release.

DACS Lock Developer's Guide

This document describes DACS Lock creation and manipulation of permission information. The DACS Lock package is made available as a supplement to ETA so applications can properly create any necessary permission data for content they provide.

ANSI Developer's Guide

This document describes ANSI Page encoding and decoding. The ANSI package is made available as a supplement to ETA so, if needed, an application can provide or consume ANSI Page based content.
