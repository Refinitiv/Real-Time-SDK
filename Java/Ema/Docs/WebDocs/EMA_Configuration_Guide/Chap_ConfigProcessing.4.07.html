Use of the Correct Order in the XML Schema
Consider the following snippet from an EmaConfig.xml (only those parts needed for the example are included). In this snippet, the application creates a consumer with a Name of Consumer_1.
 
 
<ConsumerGroup>
    <ConsumerList>
        <Consumer>
            <Name value="Consumer_1"/>
        </Consumer>
    </ConsumerList>
</ConsumerGroup>
 
Now assume that the following was not included in EmaConfig.xml:
 
 
<Directory>
    <Name value="Directory_1"/>
 
In this case, the EMA application relies on its hard-coded behavior.
However, if the snippet were configured in either of the following configurations, the EMA application would revert to its default behaviors because its parameters are not in the correct order (i.e., the Name parameter needs to be contained in a Directory component entry):
•
Configuration 1:
 
 
<DirectoryGroup>
    <Name value="Name"/>
    <DirectoryList>
        ...
 
•
Configuration 2:
 
 
<DirectoryGroup>
    <DirectoryList>
        <Name value="Name"/>
        <Directory>
        ...
