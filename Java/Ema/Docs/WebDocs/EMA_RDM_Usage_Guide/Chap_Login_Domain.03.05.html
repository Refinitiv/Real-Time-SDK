Login Refresh Message
A Login refresh message is encoded using RefreshMsg and sent by OMM interactive provider applications. This message is used to respond to a Login Request message after the user’s Login is accepted. An OMM provider can use the Login request information to authenticate users with DACS. After authentication, a refresh message is sent to convey that the login was accepted. If the login is rejected, a Login status message should be sent as described in Section 3.2.7.
