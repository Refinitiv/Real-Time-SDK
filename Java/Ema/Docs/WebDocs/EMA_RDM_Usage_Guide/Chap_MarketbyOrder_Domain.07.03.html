Market By Order Refresh Message Use
A Market By Order refresh message is encoded using RefreshMsg and sent by OMM interactive provider and OMM non-interactive provider applications. A Market By Order refresh may be sent in multiple parts. It is possible for update and status messages to be delivered between parts of a refresh message, regardless of whether the request is streaming or non-streaming.
 
Note: The provider should send the Name and ServiceName only in the first Refresh response message. However if MsgKeyInUpdates is set to true, then the Name and ServiceName must be provided for every Refresh response message.
